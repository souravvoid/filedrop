# CLEANUP GUIDE

## 1. List of Unnecessary Files

The following files should be removed as they are redundant, temporary, or outdated:

- `dependency-reduced-pom.xml` - Generated by Maven shade plugin, should not be in version control
- `target/` directory - Maven build output, should be excluded from version control
- `.idea/` - IntelliJ IDEA project files, IDE-specific configuration
- `.vscode/` - VS Code settings, IDE-specific configuration
- `p/` - Empty directory with no purpose
- `packaging/peerlink-cli.sh` - Unused CLI script that references non-existent `PeerLink.java`

## 2. What Should Be Kept

Essential files that must be preserved:

### Documentation
- `README.md` - Main project documentation
- `BUILD.md` - Build instructions
- `DESIGN.md` - UI/UX design specifications
- `INSTALL.md` - Installation guide

### Source Code
- All Java source files in `src/main/java/`
- All resources in `src/main/resources/`
- `pom.xml` - Maven build configuration

### Build and Packaging
- `build.sh` - Primary build script
- `create_icons.sh` - Icon generation script
- `Makefile` - Build automation
- `.github/workflows/release.yml` - CI/CD pipeline

### Configuration
- `com.peerlink.PeerLink.yaml` - Flatpak packaging configuration
- `packaging/` directory contents (except `peerlink-cli.sh`)

### Key Changes from Current Structure

1. **Remove IDE-specific directories**: `.idea/` and `.vscode/` should not be in version control
2. **Remove generated files**: `dependency-reduced-pom.xml` and `target/` directory
3. **Organize scripts**: Move build scripts to `scripts/` directory
4. **Remove unused scripts**: `packaging/peerlink-cli.sh` is not functional
5. **Standardize directory names**: Use consistent naming conventions
6. **Remove empty directories**: `p/` serves no purpose

### .gitignore Recommendations

Add the following to `.gitignore`:
```
# Maven
target/
!target/maven-archiver/pom.properties

# IDE
.idea/
.vscode/
*.iml

# OS
.DS_Store
Thumbs.db

# Build
dependency-reduced-pom.xml
build.log
```

This cleanup will result in a cleaner, more professional repository that focuses on essential source code and build artifacts while excluding generated and IDE-specific files.
