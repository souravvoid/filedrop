# PeerLink (Filedrop)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Java Version](https://img.shields.io/badge/Java-21%2B-blue.svg)](https://www.oracle.com/java/technologies/downloads/)
[![Build Status](https://img.shields.io/badge/Build-Passing-brightgreen.svg)](#)

PeerLink (Filedrop) is a lightweight, high-performance, serverless, peer-to-peer (P2P) file transfer application built with Java 21, JavaFX, and modern cryptography. It allows users to securely share files of any size over a Local Area Network (LAN) with zero configuration and zero external dependencies.

---

## Key Features

- **Serverless LAN P2P**: Direct socket connections between peers on the same network.
- **Enterprise-Grade Cryptography**: ECDH (secp256r1) key exchange with AES-256-GCM authenticated encryption for maximum security.
- **Path Traversal Protection**: Complete file system security using filename sanitization during receiving.
- **Vibrant Aurora Borealis UI**: High-fidelity, modern Dark Mode interface designed with smooth hover effects, micro-animations, and visual status feedback.
- **Virtual Threads Concurrency**: Engineered using Java 21 virtual threads for highly scalable, non-blocking I/O.
- **Platform-Native Packaging**: Builds into portable native formats including Linux AppImage, DEB, RPM, and Windows EXE/MSI installers.

---

## Requirements

- **Java Development Kit (JDK)**: Version 21 or later.
- **Apache Maven**: Version 3.6 or later.

## Environment Variables

PeerLink runs out-of-the-box without requiring any mandatory environment variables. However, you can configure the runtime behavior using standard JVM and display parameters:

| Variable | Description | Default / Recommended Value |
|----------|-------------|----------------------------|
| `JAVA_HOME` | Path to your JDK 21+ installation directory | `/usr/lib/jvm/java-21-openjdk` |
| `PATH` | System execution path (must include `java` and `mvn`) | `PATH=$JAVA_HOME/bin:$PATH` |
| `DISPLAY` | X11 display identifier (required for Linux GUI) | `:0` |
| `WAYLAND_DISPLAY` | Wayland display identifier (for Wayland sessions) | `wayland-0` |

## Quick Setup & Run (Development)

Clone the repository and run the application locally using Maven:

```bash
# Clone the repository
git clone https://github.com/souravvoid/filedrop.git
cd filedrop

# Build and run immediately
mvn clean compile exec:java -Dexec.mainClass="com.peerlink.ui.Launcher"
```

## Build & Packaging Instructions

PeerLink uses Maven profiles to compile and assemble native application packages.

### 1. Build Fat Shaded JAR

To build a standalone, executable JAR that can be run on any platform with a JDK 21+ environment:

```bash
mvn clean package -DskipTests
```
* **Output:** `target/peerlink-1.0.0-shaded.jar`
* **Execution:** `java -jar target/peerlink-1.0.0-shaded.jar`

### 2. Linux Packaging (DEB, RPM, and AppImage)

On Linux systems, package build profiles are activated automatically:

```bash
mvn clean package
```

This will produce three distribution packages under `dist/linux/`:
- **Debian/Ubuntu Package**: `PeerLink-1.0.0.deb` (Install using `sudo dpkg -i dist/linux/PeerLink-1.0.0.deb`)
- **Fedora/RHEL Package**: `PeerLink-1.0.0.rpm` (Install using `sudo dnf install dist/linux/PeerLink-1.0.0.rpm`)
- **Portable AppImage**: `PeerLink-1.0.0-x86_64.AppImage` (Run directly with `./dist/linux/PeerLink-1.0.0-x86_64.AppImage` after running `chmod +x`)

### 3. Windows Packaging (EXE and MSI)

On Windows systems, build installers with:

```cmd
mvn clean package
```
* **Requirements**: WiX Toolset must be installed.
- **Executable Installer**: `dist/windows/PeerLink-1.0.0.exe`
- **MSI Enterprise Package**: `dist/windows/PeerLink-1.0.0.msi`

## Security Architecture

PeerLink implements a zero-trust design for local P2P transfers:

1. **Invite Code Generation**: The receiver generates a cryptographically encoded invitation containing their local IP, active transfer port, and public key fingerprint.
2. **ECDH Key Exchange**: Upon connection, a unique ephemeral Elliptic Curve Diffie-Hellman (secp256r1) key pair is generated by both parties to derive a shared secret key.
3. **AES-256-GCM AEAD Encryption**: Every metadata and file chunk is encrypted using AES-256-GCM with distinct IVs.
4. **Filename Sanitization**: Output filenames are aggressively stripped of directory separators (`/`, `\`, `..`) to ensure no path traversal vulnerability exists.
5. **Memory Bomb Mitigation**: Incoming packet size thresholds (e.g., metadata capped at 8KB) are strictly validated before buffering to prevent memory allocation denial-of-service attacks.
