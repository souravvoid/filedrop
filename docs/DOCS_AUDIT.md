## BUILD_COMMANDS.md — ⚠️ Issues Found

| Line | Issue |
|------|-------|
| "Build DEB — `mvn clean package -DskipLinux=false`" | ❌ Wrong flag. Correct: `-DskipDeb=false` |
| "Build RPM — `mvn clean package`" | ❌ Auto-activation only on Debian; on Fedora `skipRpm` auto-enables |
| "Build AppImage — `mvn clean package`" | ❌ AppImage requires `-DskipAppImage=false` or `-Pappimage` |
| "Output: `dist/linux/PeerLink-1.0.0-x86_64.AppImage`" | ❌ jpackage `app-image` type produces a directory, not `.AppImage` |
| "Run JAR: `java -jar target/peerlink-1.0.0-shaded.jar`" | ❌ Fails without `--module-path` for JavaFX |

**Correct JAR run command:**
```bash
java --module-path ~/.m2/repository/org/openjfx/... \
     --add-modules javafx.controls,javafx.fxml \
     -jar target/peerlink-1.0.0-shaded.jar
# OR use Maven:
mvn javafx:run
```
